## Questions?

Note:
Backup talking points for quiet Q&A:

- **Kotlin / Scala / other JVM languages?** Works. JDWP doesn't care about source language; JDI exposes line numbers from `.class` files.
- **Performance impact on the target JVM?** JDWP itself adds <1% overhead when no breakpoints are armed. Breakpoint hits cost ~1ms per event. Logpoints are cheaper than printlns. *Field* watchpoints can be expensive on hot fields — narrow with filters, prefer short-lived sessions.
- **Security?** JDWP is unauthenticated. Don't expose port 5005 to the internet. Bind to localhost in CI. README has a security & trust section.
- **What about virtual threads (Loom)?** Supported by JDI starting from JDK 21. We expose them via `get_threads`. Stack frames for mounted virtual threads look like platform threads.
- **Roadmap?** Field watchpoints, chained breakpoints, marked instances, log-only exception breakpoints all landed in 2.0. Recent: 2.1.0 audit fixes + transport-loss envelopes, 2.1.1 log file no longer dirties user CWD. Next: ?
- **Where do I file bugs?** GitHub issues on `FgForrest/mcp-jdwp-java`.
- **OSS or commercial?** MIT license. Built at FG Forrest, open to external contribution. Original upstream attribution: [Nicolas Vautrin](https://github.com/NicolasVautrin/mcp-jdwp-java).
