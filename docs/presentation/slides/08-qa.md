Note:
Backup Q&A bullets:
- Kotlin / Scala / other JVM languages — works; JDWP doesn't care about source language
- Perf on target JVM — <1% when nothing armed; ~1 ms per BP hit; logpoints cheaper than printlns
- Field watchpoints can be expensive on hot fields — filter / short sessions
- Security — JDWP is unauthenticated; don't expose 5005 to the internet; bind localhost in CI
- Virtual threads — supported by JDI from JDK 21; we expose them via `get_threads`
- Roadmap — recently shipped: field watchpoints, chains, marks, exception logpoints, audit fixes
