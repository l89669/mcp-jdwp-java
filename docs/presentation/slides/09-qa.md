## Questions?

Note:
Backup talking points if Q&A is quiet:
- What about Kotlin / Scala / other JVM languages? — Works. JDWP doesn't care about source language; JDI exposes line numbers from the .class files.
- Performance impact on the target JVM? — JDWP itself adds <1% overhead when no breakpoints are armed. Breakpoints hit cost ~1ms per event. Logpoints are cheaper than printlns.
- Security? — JDWP is unauthenticated. Don't expose port 5005 to the internet. Bind to localhost in CI.
- Roadmap? — Field watchpoints just landed. Next: better support for virtual threads, conditional logpoint rate-limiting.
